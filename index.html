rebase [-i]

know it. love it.

Can:

  • move commits to a new base point
  • interactive - re-order, drop, squash commits
  • not everyone is a fan!
  • replace stash!
  • Drop commits with secrets (if you catch before you push)

remember: only on your local commits

rebase [-i]

use it for good

Rebase is powerful!

Use it wisely

(e.g. removing secrets)

When creating a PR, rebase on latest point of branch.

Keep it simple for the reviewer/maintainer

On that note, also favour multiple PRs over on large PR with several unrelated changes
